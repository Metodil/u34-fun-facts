## Branching strategy
For this repo I'm using Trunk Based Development (TBD) flow strategy, because:
> - Branch are small and self sufficiental;
> - Keep close to the `main` branch;
> - Reduce merge conflicts
> - Fast feedback loops for the CI part;
> - Quick release.
